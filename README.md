# Innovation Hub Dashboard

A **Shiny dashboard** (built with [golem](https://thinkr-open.github.io/golem/)) for the World Bank's [Poverty and Inequality Platform (PIP)](https://pip.worldbank.org/) Innovation Hub. It showcases experimental poverty estimation methods developed by the DECDG team and provides a semantic search interface for exploring related research documents.

**Live deployment**: Two internal World Bank Posit Connect servers (see [Deployment](#deployment)).

## Features

The app has three pages, controlled by a custom tab-switching nav header:

| Page | Description |
|---|---|
| **Home** | Landing page with banner text and a 2×2 grid of method cards. Clicking a card navigates to Deep Dives with that method preselected. |
| **Deep Dives** | Interactive dashboard for exploring 4 experimental methods: Welfare conversion (DM), Household allocation (STB), Subnational definition (SN), and NA–Survey gap adjustment (YK). Includes country selectors, plotly charts, and downloadable data. |
| **Research Repository** | A Vue 3 / Vuetify semantic search app embedded via `<iframe>`. Performs in-browser AI-powered search over World Bank research documents using [transformers.js](https://huggingface.co/docs/transformers.js). |

## Getting Started

### Prerequisites

- **R ≥ 4.3** (built on 4.3.1)
- **renv** for dependency management

### Setup

```r
# 1. Clone the repo
# 2. Open InnovationHubDashboard.Rproj in RStudio/Positron

# 3. Restore dependencies
renv::restore()

# 4. Ensure data files are present:
#    - data/*.rda    (committed to git)
#    - data/*.xlsx   (gitignored — obtain from a team member)

# 5. Run the app
golem::run_dev()
# OR: source("dev/run_dev.R")
```

### Required data files (gitignored)

These `.xlsx` metadata files must be present in `data/` but are not tracked in git:

- `dm_metadata.xlsx`
- `stb_metadata.xlsx`
- `sn_metadata.xlsx`
- `yk_metadata.xlsx`

Obtain them from the current maintainer or regenerate from the source `.dta` files.

## Data Pipeline

### How data flows

```
Source .fst / .dta files  →  data-raw/*.R scripts  →  data/*.rda  →  app_server.R loads at runtime
```

### Updating data

1. Obtain updated source data files (`.fst` or `.dta` format) from the DECDG team.
2. Edit the corresponding script in `data-raw/` to point to the new file.
3. Run the script to regenerate the `.rda`:

```r
source("data-raw/d_dm.R")  # regenerates data/d_dm.rda
```

4. Update the corresponding `.xlsx` metadata file if method descriptions changed.
5. Restart the app to verify.

> **Note:** The `data-raw/` scripts contain hardcoded absolute paths to source data (e.g., `C:/Users/wb612474/...`). Update these paths to match your local environment.

### Data objects

| Object | Source script | Description |
|---|---|---|
| `d_dm` | `data-raw/d_dm.R` | Welfare conversion (Dollar a day → local methods) |
| `d_stb` | `data-raw/d_stb.R` | Household allocation (survey-to-budget) |
| `d_sn` | `data-raw/d_sn.R` | Subnational poverty definition |
| `d_yk` | `data-raw/d_yk.R` | NA–Survey gap adjustment |
| `dcol` | `data-raw/dcol.R` | Colombia-specific cross-method comparison |

## Research Repository

The Research Repository tab embeds a **standalone Vue 3 / Vuetify app** inside an `<iframe>`. It has no R backend — it runs entirely in the user's browser.

### How it works

1. On page load, it fetches a pre-computed **embeddings JSON** (~MB file containing document titles + AI vector fingerprints).
2. When the user searches, it loads an **AI model** ([GIST-small-Embedding-v0](https://huggingface.co/avsolatorio/GIST-small-Embedding-v0)) directly in the browser via [transformers.js](https://huggingface.co/docs/transformers.js) + WebGPU.
3. It converts the query to a vector and finds the most similar documents via dot-product similarity.
4. When a result is clicked, it fetches **document metadata** from a remote endpoint.

### File locations

| File | Purpose |
|---|---|
| `inst/app/www/research_repo/ids-doc.html` | The patched Vue app (served in the iframe) |
| `inst/app/www/research_repo/css/*.min.css` | Vuetify theme styles (compiled from SCSS) |
| `inst/app/www/research_repo/data/*.json` | Pre-computed document embeddings |
| `ai-for-data-blog/` | Full source repo (SCSS source, Python scripts, original HTML) |

### What loads locally vs. remotely

| Resource | Source |
|---|---|
| CSS (`universal.min.css`, `ids.min.css`) | **Local** — `www/research_repo/css/` |
| Embeddings JSON | **Local** — `www/research_repo/data/` |
| Vue 3, Vuetify, Material Icons | **Remote** — CDN (jsdelivr) |
| AI model (GIST-small-Embedding-v0) | **Remote** — Hugging Face |
| Document metadata (per-document JSON) | **Remote** — `avsolatorio.github.io/ids-metadata/` |

### Updating the Research Repository

#### UI / behavior changes

Edit `inst/app/www/research_repo/ids-doc.html` directly. It's plain HTML + inline JavaScript (Vue 3 composition API). No build step required.

#### CSS / style changes

The `.min.css` files are compiled from SCSS source in `ai-for-data-blog/semantic-search/scss/`. Two options:

- **Quick**: Edit the `.min.css` files directly for minor tweaks.
- **Proper**: Install a SCSS compiler and rebuild:

```bash
npm install -g sass
sass ai-for-data-blog/semantic-search/scss/ids.scss inst/app/www/research_repo/css/ids.min.css --style=compressed
```

#### Adding or removing papers (document corpus)

This is the most involved update. The embeddings JSON is pre-computed — each document has a vector fingerprint. To update:

1. **Update the embeddings**: Run the Python embedding generation script with the updated document list. This requires:
   - A Python environment with the `sentence-transformers` and `optimum` packages
   - The `ai-for-data-blog/scripts/generate_wdi_embeddings.py` script (may need adaptation)
   - The Hugging Face model `avsolatorio/GIST-small-Embedding-v0`

2. **Copy the output** JSON to `inst/app/www/research_repo/data/`.

3. **Update document metadata**: Per-document metadata is served from `https://avsolatorio.github.io/ids-metadata/`. You need write access to that repository to add/update metadata JSON files.

> **Important**: Get a detailed runbook from the original developer (avsolatorio) covering the exact Python dependencies, input format, and the `ids-metadata` repo update process.

## Updating & Maintenance

### Adding a new experimental method

1. Create the data processing script in `data-raw/` → generates a new `.rda`.
2. Add the `.rda` to `data/` and update `app_server.R` to load it.
3. Add a new option to the method picker in `mod_interactive_dashboard.R`.
4. Write the corresponding plotting function in a new `fct_plot_*.R` file.
5. Add a method card to `mod_home.R`.
6. Add the landing image to `inst/app/www/`.

### Changing the header / navigation

- **CSS**: All header/nav/card styles are in `inst/app/www/header.css`.
- **Nav items**: Defined in `R/app_ui.R` (the `tags$header(...)` block).
- **Tab switching logic**: `R/app_server.R` — the `switch_nav()` helper and `observeEvent` blocks.

### Adding a banner above the Research Repository

A placeholder `div.research-repo-banner` already exists in `app_ui.R`. Add content there:

```r
tags$div(
  class = "research-repo-banner",
  tags$p("Your banner content here")
)
```

Style it in `header.css` under `.research-repo-banner`.

### Dependency management

```r
# After adding/removing packages
renv::snapshot()

# To check for issues
renv::status()
```

Always commit `renv.lock`. Never commit `renv/library/`.

## Key Technical Decisions

| Decision | Rationale |
|---|---|
| **golem framework** | Standard R package structure for Shiny apps. Enables `devtools`/`testthat` workflows. |
| **iframe for Research Repository** | The Vue app uses Vuetify CSS + Vue 3 JS that would conflict with Shiny's Bootstrap. iframe provides complete isolation. |
| **Client-side AI search** | The semantic search runs entirely in the browser (no R/Python backend needed). Uses WebGPU for model inference. |
| **`require()` at runtime** | Several packages are loaded via `require()` in `app_server.R` for their side-effects. Acknowledged tech debt — future refactor should use explicit `pkg::fn()` calls. |
| **Hardcoded country lookup** | `app_server.R` contains a 20-country lookup table. If the country set changes, update it there. |
